# Telegram-бот для nail-мастера

Демонстрационный бот на `aiogram 3` для записи клиентов на маникюр. Его можно показывать в холодных рассылках как готовый продукт: есть запись по шагам, портфолио, отзывы, FAQ, контакты и скрытая панель мастера.

## Что умеет

- `/start` с коротким приветствием и вопросом «откуда узнали о нас».
- Главное меню с разделами: запись, примеры работ, отзывы, FAQ, контакты, услуги.
- Запись через FSM: услуга → дата → время → имя и телефон → подтверждение.
- Inline-календарь на 30 дней вперед: прошедшие, нерабочие, закрытые и занятые даты недоступны.
- Проверка слота при подтверждении, чтобы два клиента не заняли одно время.
- Уведомление мастеру в личку после новой записи.
- Портфолио с категориями и альбомами фотографий.
- Отзывы: просмотр опубликованных, сбор новых, модерация мастером.
- FAQ с inline-вопросами.
- Контакты с кнопками на Instagram и карту.
- `/admin`: записи на сегодня, записи на неделю, блокировка дат, статистика источников.

## Быстрый запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

После этого откройте `.env` и заполните:

```dotenv
BOT_TOKEN=токен_от_BotFather
MASTER_CHAT_ID=telegram_id_мастера
ADMIN_IDS=telegram_id_мастера
```

Если не знаете свой `telegram_id`, сначала запустите бота только с `BOT_TOKEN`, отправьте ему `/myid`, затем вставьте показанный ID в `MASTER_CHAT_ID` и `ADMIN_IDS`.

Запуск:

```powershell
python -m app.main
```

## Где менять контент

- `data/services.json` — услуги, цены, длительность.
- `data/portfolio.json` — категории и фото работ.
- `data/reviews.json` — стартовые отзывы и новые отзывы после модерации.
- `data/faq.json` — вопросы и ответы.
- `data/bookings.json` — подтвержденные записи.
- `data/blocked_dates.json` — закрытые даты.
- `data/leads.json` — ответы на вопрос «откуда узнали».

## Админ-панель

Команда `/admin` доступна только ID из `ADMIN_IDS` и `MASTER_CHAT_ID`.

В панели можно:

- посмотреть записи на сегодня;
- посмотреть записи на ближайшие 7 дней;
- закрыть или снова открыть дату для записи;
- посмотреть статистику источников из стартового опроса.

## Примечания для демо

Хранилище сделано на JSON-файлах, чтобы проект было легко показать и быстро перенести. Для продакшена стоит заменить `JsonStore` на PostgreSQL или SQLite, добавить напоминания за 24 часа, перенос записей клиентом и отдельную CRM-страницу мастера.
